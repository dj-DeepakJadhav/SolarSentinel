# Solar Sentinel — 2:50 Demo Script

Use a public YouTube video with voiceover. Keep the interface visible; never cut to a fake mock-up or an unlabelled data state.

| Time | Screen | Voiceover |
| --- | --- | --- |
| 0:00–0:12 | Landing: NASA SDO Sun, `LIVE`, one Kp pod | “Space weather is real, but most people only meet it as a specialist feed. Solar Sentinel makes the Sun–Earth system something a class can investigate together.” |
| 0:12–0:25 | Trigger the Sun → spacecraft transition | “The Sun texture is the latest available NASA Solar Dynamics Observatory image. This flight is illustrative: it gives the solar wind story a memorable shape without pretending to model every particle.” |
| 0:25–0:43 | Arrive at Earth; show shield, aurora, Kp, source drawer | “Here the product separates what was observed from NOAA’s official forecast and from our own educational scenario. Every data layer carries a timestamp and source link.” |
| 0:43–1:00 | Open Mission → Locate; search a city such as `Tokyo`; select a result | “A learner can search for a city anywhere in the world. The globe rotates its real geographic surface and beacon toward that city, while the magnetic shield stays Sun-facing.” |
| 1:00–1:22 | Student mode → Ask: `What am I seeing?` | “In Student mode, GPT turns the current, validated snapshot into four short truth layers: observed data, NOAA forecast, scenario, and uncertainty. Source notes are attached by the app, not invented by the model.” |
| 1:22–1:45 | Toggle Teacher mode → Ask: five-minute discussion | “Teacher mode changes the activity, not the science. It gives a classroom sequence: observe the scene, compare observation with forecast, identify the simulation, and discuss uncertainty.” |
| 1:45–2:05 | Mission → What if?; move from Quiet to Strong then return to Quiet | “The scenario interaction is where students can ask ‘what changes if conditions strengthen?’ The on-screen disclaimer remains: this is an educational simulation, never a prediction or operational advisory.” |
| 2:05–2:26 | Mission → Replay; scrub May 2024 timeline | “For a reliable dramatic lesson even on a quiet day, the replay uses NASA DONKI’s archived May 2024 geomagnetic-storm event. It is clearly historical and linked back to its catalogue entry.” |
| 2:26–2:43 | Open How we know; show source cards and cache state if available | “Live, delayed, cached, replay, and unavailable are treated as different product states. Solar Sentinel never hides when a source is stale or when it has fallen back.” |
| 2:43–2:50 | Return to full Sun–Earth view | “Solar Sentinel is a reusable model for future space-learning missions: a beautiful visual story, grounded in the sources that make it trustworthy.” |

## Required capture checklist

- Record in a browser with audio narration; keep it under 3:00.
- Show `LIVE` or `CACHED` honestly—do not edit a status label to imply live data.
- Show the source drawer, Student and Teacher modes, a city rotation, scenario disclaimer, and DONKI replay.
- In the Devpost description, state that Codex was used to build the application and GPT-5.6 powers the optional source-grounded briefing.
- Add the real Codex `/feedback` session ID to the submission after recording; do not place it in this repository.
